## 🔧 当前开发环境说明

### Mock API vs 真实后端

**当前状态**：使用 `mock-server.cjs` 提供Mock数据，适用于前端开发。

**切换到真实后端**：
1. 启动PostgreSQL和Redis
2. 进入 `shelf-erp-server/` 目录
3. 运行 `npm run start:dev`
4. 修改 `vite.config.ts` 中的proxy目标为 `http://localhost:3000` → `http://localhost:3000`

### 添加新Mock接口

编辑 `mock-server.cjs`，在文件末尾的"兜底"路由前添加：

```javascript
// 示例：添加新接口
app.get('/api/m08/purchases', (req, res) => {
  res.json(ok({
    items: [
      { id: '1', code: 'PO-2024-001', supplierName: '测试供应商', totalAmount: 30000, status: 'pending' }
    ],
    total: 1
  }));
});
```

## 🚨 常见问题

### 登录后页面空白
- 检查 `mock-server.cjs` 是否运行（端口3000）
- 检查浏览器控制台（F12）是否有错误
- 确认Vite proxy配置正确（`vite.config.ts`）

### `Cannot read properties of undefined (reading 'reduce')`
- 通常是DashboardPage的数据防御不足
- 已修复：所有store订阅都加了 `?? []` 防御

### E2E测试失败
- 确认Vite开发服务器运行在5173端口
- 确认使用绝对URL（`http://localhost:5173/shelf-erp/...`）
- 检查 `.or()` 链是否加了 `.first()`
