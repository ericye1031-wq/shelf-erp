## 🔧 六、关键技术要点

### 6.1 Vite 配置

- `base: '/shelf-erp/'` — 所有路由需带此前缀
- GitHub Pages部署需设置正确base路径
- E2E测试中 `page.goto()` 必须使用绝对URL：`http://localhost:5173/shelf-erp/...`

### 6.2 E2E测试 Mock 方案

- **不使用MSW**（在Playwright环境中无法正确注册）
- 使用 `e2e/helpers/mock-api.ts` 中的 `page.route('**/api/**', handler)`
- `setupTestEnv(page)` — 自动设置认证状态 + API mock
- `setupApiMock(page)` — 仅设置API mock（用于登录测试）

### 6.3 BOM计算引擎

- **前端**: `src/utils/bomCalculator.ts`
- **后端**: `shelf-erp-server/src/m04/bom-calculator/`
- **数据源**: `src/data/material-profiles.ts`（真实型材重量数据）
- **公式**: 数量 × 长度 × 单位重量 × 单价 × (1 + 损耗率)

### 6.4 认证机制

- JWT Token认证（Access Token + Refresh Token）
- Zustand持久化到localStorage
- 自动token刷新拦截器

## ⚠️ 七、重要注意事项

1. **TypeScript类型检查**：构建前务必运行 `npm run build:check`，确保0 errors
2. **E2E测试URL**：必须使用绝对路径，不能使用相对路径
3. **Playwright locator**：`.or()` 链必须加 `.first()` 避免strict mode violation
4. **后端注册**：所有模块已注册到 `AppModule`，新增模块需手动注册
5. **数据库连接**：首次运行需执行TypeORM迁移或手动建表
