## Step 1：安装 Docker Desktop（仅需一次）

1. 打开浏览器访问：https://www.docker.com/products/docker-desktop/
2. 下载 **Docker Desktop for Windows**（约 550MB）
3. 运行安装包，全程下一步（勾选 WSL2 集成）
4. 安装完成后**重启电脑**
5. 重启后 Docker Desktop 会自动启动（任务栏有鲸鱼图标）

> ⚠️ Windows 家庭版需要先开启 Hyper-V 或 WSL2，Docker Desktop 安装时会自动引导。

### 验证安装成功

```bash
docker --version
# 输出类似：Docker version 24.x.x, build xxxxx
```

---

## Step 2：启动后端（Docker 装好后）

**最简方式**：双击桌面上的 `启动后端.bat`

脚本会自动完成：
1. 启动 PostgreSQL 16 + Redis 7 容器
2. 等待数据库就绪（15秒）
3. 安装后端 npm 依赖
4. 初始化种子数据（首次运行）
5. 启动 NestJS 开发服务器

**手动命令（如果脚本有问题）**：

```bash
# 进入后端目录
cd "C:\Users\DELL\Desktop\货架erp 260620\shelf-erp-server"

# 启动数据库容器
docker-compose up -d

# 等待15秒后安装依赖
npm install

# 初始化数据（首次运行）
npm run seed

# 启动后端
npm run start:dev
```

### 验证后端启动成功

浏览器打开：http://localhost:3000/api/docs
> 应该看到 Swagger API 文档界面

## 注意事项

### vite.config.ts 不需要改动

前端 proxy 已配置指向 `localhost:3000`，NestJS 也跑在 3000 端口，自动接管。

### 如果端口 5432 被占用

编辑 `shelf-erp-server/docker-compose.yml`：
```yaml
ports:
  - "5433:5432"  # 改成5433
```
同时更新 `shelf-erp-server/.env.local`：
```env
DATABASE_PORT=5433
```

### 如果 seed 报错（非首次运行）

种子数据已存在时会有主键冲突，属正常现象，不影响后端启动。
