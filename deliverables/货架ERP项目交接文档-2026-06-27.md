# 货架ERP系统 — 项目交接文档

> 版本：v0.1.0 | 日期：2026-06-27 | 状态：开发中（80%模块完成）

---

## 一、项目概述

货架ERP系统是专为货架制造企业设计的一体化管理平台，覆盖从销售、设计、报价、生产到售后全流程。系统采用前后端分离架构，目前已完成17个核心模块的后端与前端开发（M01-M17），剩余3个AI模块（M18-M20）待开发。

---

## 二、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **后端框架** | NestJS 10.x | TypeScript企业级后端框架 |
| **ORM** | TypeORM 0.3.x | 数据库ORM，实体映射 |
| **数据库** | SQLite (better-sqlite3) | 轻量嵌入式数据库，无需Docker |
| **认证** | Passport + JWT | 无状态Token认证 |
| **前端框架** | React 18.x + Vite 5.x | SPA前端 |
| **UI组件** | MUI 5.x + Tailwind CSS | Material风格 + 样式工具 |
| **状态管理** | Zustand 4.x | 轻量状态管理 |
| **HTTP客户端** | Axios | 前端API请求 |
| **图表** | Recharts | 数据可视化 |
| **日期处理** | Day.js | 轻量日期库 |

cd shelf-erp-server && npm run seed

npm run start:dev

## 九、SQLite兼容性要点（重要）

开发过程中踩过的坑，后续开发务必注意：

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| `@CreateDateColumn` | 不能指定 `type: 'datetime'` | TypeORM自动处理，不指定type |
| `@Column({ length: X })` | SQLite不支持length约束 | 显式指定 `type: 'text'` |
| 枚举字段 | SQLite不支持TypeORM的`enum`类型 | 用 `type: 'text'`，属性改为 `string` |
| `nullable`字段 | seed中`repo.create()` | 用 `undefined` 而非 `null` |
| `NOW()` | PostgreSQL语法 | SQLite不支持，不使用default日期函数 |
| `pageSize` DTO | GET无参数时报400 | 必须加 `@IsOptional()` |
| `repo.save()` | 返回 `Entity[]` | 用 `as unknown as Promise<Entity>` |
| `repo.findOne()` | 返回 `Entity | null` | 返回类型必须 `Promise<Entity | null>` |
| 变量名冲突 | 不同模块的repo变量名 | 种子数据中用模块前缀（如 `m17ReportRepo`） |
| 标识符冲突 | TypeORM `DataSource` vs M17实体 | 导入时用别名 `DataSource as M17DataSourceEntity` |
| 枚举值赋值 | TypeScript类型不匹配 | 种子数据中用 `as any` 断言 |
